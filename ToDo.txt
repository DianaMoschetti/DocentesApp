Reglas rápidas de dependencias (para que quede prolijo)

Domain ← Application ← API
Domain ← Data ← API

Idealmente:

Application no depende de Data.

Data implementa interfaces de Application.

API referencia a ambos para configurar DI.


-- identity para usuarios


MIGRACIONES 
en la raiz del proy
dotnet ef migrations add InitialCreate --project DocentesApp.Data --startup-project DocentesApp.API

luego la aplico
dotnet ef database update --project DocentesApp.Data --startup-project DocentesApp.API
